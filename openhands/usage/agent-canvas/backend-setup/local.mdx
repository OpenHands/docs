title: Local Backend
description: The default backend that ships with Agent Canvas.
---

When you run `agent-canvas`, a local backend starts automatically. No extra setup is needed.

## What You Get

The `agent-canvas` command starts:

- An **agent server** on `127.0.0.1:18000`
- An **automation backend** for cron and event-driven workflows
- An **ingress proxy** on `127.0.0.1:8000` that routes between them
- A **static frontend** served through the same ingress

Agent Canvas seeds a default backend entry pointing to this stack. It appears in the backend switcher as your local backend.

## When to Use It

The local backend is a good fit when you want to:

- Run agents entirely on your own machine
- Test configuration changes quickly
- Experiment with skills, MCP servers, or automations before deploying elsewhere

## Accessing from Other Devices on Your Network

By default, Agent Canvas only listens on `127.0.0.1` (localhost). To access it from another device on your LAN — for example a phone or tablet — start in public mode:

```bash
LOCAL_BACKEND_API_KEY=<choose-a-strong-secret> agent-canvas --public
```

The `--public` flag requires anyone who opens the UI to enter the API key before interacting with the agent. Without it, the key is auto-injected into the frontend, which is convenient for solo local use but unsafe when others can reach the host.

<Warning>
  The agent server can execute shell commands and access the filesystem. Only expose it on your network if you trust every device that can reach it.
</Warning>
