---
title: Model Selection Tool
description: Let the agent dynamically switch between saved LLM profiles during an Agent Canvas task.
---

The model selection tool lets the agent switch a running Agent Canvas conversation to another saved LLM profile when the
next step would benefit from a different model. It is useful when you want OpenHands to choose the best available profile
for each task phase without manually pausing the task.

In the Agent SDK, this capability is exposed as the built-in `SwitchLLMTool`, which produces `switch_llm` tool calls.
Agent Canvas displays those tool calls and observations in the conversation timeline so you can see when and why the
model changed.

<Note>
The agent can only switch to profiles that already exist. Create profiles in
[Language Model (LLM) Settings](/openhands/usage/settings/llm-settings) before asking the agent to select models
dynamically.
</Note>

## Example Profiles

Dynamic switching works with any saved profiles. The profile names in this guide are examples only; replace them with
models and profile names that exist in your Agent Canvas environment.

| Example Profile | Example Use |
| --- | --- |
| `claude-opus-4-7` | The task needs stronger frontend design or visual polish. |
| `gpt-5.5` | The task needs careful instruction following, planning, or review. |
| `minimax-m2.7` | The task is routine implementation where lower cost matters. |

## Dynamic Switching Workflow

1. Create the profiles you want the agent to choose from. This example uses `claude-opus-4-7`, `gpt-5.5`, and
   `minimax-m2.7`, but any saved profiles can be used.
2. Start the conversation with a default profile, such as the profile you normally use for routine work.
3. Ask OpenHands to use specific profiles for different phases of the task. Use your own profile names in the prompt if
   you are not using the example names:

   ```text
   Implement a simple web page on the features of OpenHands with Claude Opus 4.7, and then switch to GPT-5.5 and review the code.
   ```

4. Watch Agent Canvas for `Switch LLM profile` tool events. The tool call includes the requested `profile_name` and the
   agent's reason for switching.
5. Review the final answer after the agent switches to the requested review profile and completes the code review.

![Agent Canvas showing example switch_llm tool calls that move a task between saved profiles](/openhands/static/img/model-selection-tool-agent-canvas.png)

## How the Tool Behaves

- The current model decides to call the tool and provides a short reason.
- The switch takes effect on the next LLM call after the tool succeeds.
- Conversation history, files, and task state are preserved.
- If a profile name is missing or misspelled, the tool returns an error and the agent should choose a valid profile or
  ask for help.

## SDK Configuration

For custom SDK agents, include the built-in tool when you construct the agent. Replace `minimax-m2.7` with the default
profile you want the agent to start from:

```python
agent = Agent(
    llm=store.load("minimax-m2.7"),
    tools=[],
    include_default_tools=["FinishTool", "SwitchLLMTool"],
)
```

See the ready-to-run SDK example:
[examples/01_standalone_sdk/49_switch_llm_tool.py](https://github.com/OpenHands/software-agent-sdk/blob/main/examples/01_standalone_sdk/49_switch_llm_tool.py).

## When to Use Manual vs. Dynamic Switching

| Option | Choose It When |
| --- | --- |
| [`/model` command](/openhands/usage/settings/model-command) | You want to decide exactly when the model changes. |
| Model selection tool | You want the agent to choose models based on task phase and model strengths. |
