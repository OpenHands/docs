---
title: /model Command
description: Switch a running Agent Canvas conversation between saved LLM profiles from the chat input.
---

The `/model` command gives you direct control over which saved LLM profile powers the next turn of an Agent Canvas
conversation. Use it when you want to use different models for different phases of the same task, while preserving the
conversation history, files, and workspace state.

<Note>
Model switching requires saved LLM profiles. If `/model` is not suggested in the chat input, create profiles in
[Language Model (LLM) Settings](/openhands/usage/settings/llm-settings) and confirm that your backend supports profile
switching.
</Note>

## Example Profiles

Create profiles with names that are easy to type. The profile names in this guide are examples only; you can use any
models and any profile names that are available in your Agent Canvas environment.

| Example Profile | Example Use | Example Cost Pattern |
| --- | --- | --- |
| `claude-opus-4-7` | Frontend design and visual polish | Higher cost |
| `gpt-5.5` | Planning and instruction following | Balanced for complex reasoning |
| `minimax-m2.7` | Day-to-day implementation | Lower cost |

## Command Reference

- `/model` lists saved profiles inline in Agent Canvas.
- `/model <profile-name>` switches the running conversation to that profile.

Profile names must match the saved profile exactly. The switch applies to future agent steps; it does not rerun earlier
messages. The examples below use the profile names from the table above, but the same workflow works with your own saved
profiles.

## Manual Switching Workflow

1. Start the conversation with any saved profile selected. In this example, the conversation starts with `gpt-5.5`.
2. Ask OpenHands to plan the work before editing files:

   ```text
   Plan the OpenHands features page. Do not edit files yet.
   ```

3. Type `/model` in the chat input. Before you submit, Agent Canvas shows matching saved profile completions.

   ![Agent Canvas with slash-command completions after /model is typed in the chat input](/openhands/static/img/model-command-input-agent-canvas.png)

4. Send `/model` to list the available profiles in the chat.
5. Send `/model minimax-m2.7` to switch to another example profile. Replace `minimax-m2.7` with the profile you want to
   use.
6. Ask OpenHands to implement the plan:

   ```text
   Now implement the plan.
   ```

The conversation keeps its previous context, so the newly selected profile can continue from work produced by the
previous profile.

![Agent Canvas showing example /model command output that lists saved profiles and switches to another profile](/openhands/static/img/model-command-agent-canvas.png)

## Good Practices

- Use `/model` before switching if you do not remember the exact profile name.
- Switch after a clear boundary, such as after planning, before implementation, or before review.
- Keep profile names short and descriptive so they are easy to use from the chat input.
- Use specialized or higher-cost models only where their strengths matter, then switch back to the profile you prefer
  for routine work.
