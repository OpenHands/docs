---
title: /model Command
description: Switch a running Agent Canvas conversation between saved LLM profiles from the chat input.
---

The `/model` command gives you direct control over which saved LLM profile powers the next turn of an Agent Canvas
conversation. Use it when you want to plan with one model, switch to a lower-cost model for implementation, and keep the
same conversation history.

<Note>
Model switching requires saved LLM profiles. If `/model` is not suggested in the chat input, create profiles in
[Language Model (LLM) Settings](/openhands/usage/settings/llm-settings) and confirm that your backend supports profile
switching.
</Note>

## Example Profiles

Create profiles with names that are easy to type. This example uses three profiles with different strengths:

| Profile | Best For | Cost Pattern |
| --- | --- | --- |
| `claude-opus-4-7` | Frontend design and visual polish | Higher cost |
| `gpt-5.5` | Planning and instruction following | Balanced for complex reasoning |
| `minimax-m2.7` | Day-to-day implementation | Lower cost |

## Command Reference

- `/model` lists saved profiles inline in Agent Canvas.
- `/model <profile-name>` switches the running conversation to that profile.

Profile names must match the saved profile exactly. The switch applies to future agent steps; it does not rerun earlier
messages.

## Manual Switching Workflow

1. Start the conversation with the `gpt-5.5` profile selected.
2. Ask OpenHands to plan the work before editing files:

   ```text
   Plan the OpenHands features page. Do not edit files yet.
   ```

3. Send `/model` to list the available profiles in the chat.
4. Send `/model minimax-m2.7` to switch to the lower-cost daily-driver profile.
5. Ask OpenHands to implement the plan:

   ```text
   Now implement the plan.
   ```

The conversation keeps its previous context, so `minimax-m2.7` can continue from the plan that `gpt-5.5` produced.

![Agent Canvas showing the /model command listing three profiles and switching from gpt-5.5 to minimax-m2.7](/openhands/static/img/model-command-agent-canvas.png)

## Good Practices

- Use `/model` before switching if you do not remember the exact profile name.
- Switch after a clear boundary, such as after planning, before implementation, or before review.
- Keep profile names short and descriptive so they are easy to use from the chat input.
- Use higher-cost models only where their strengths matter, then switch back to a daily-driver model.
