## Overview
The CLI provides both interactive and non-interactive modes. For scripting and automation, see [Headless Mode](#headless-mode) below.

<Note>
If you're upgrading from a CLI version before release 1.0.0, you'll need to redo your settings setup as the
configuration format has changed.
</Note>

## Getting Started

<Tabs>
  <Tab title="Using uv (recommended)">
    Requires Python 3.12+ and uv installed.

    **Install OpenHands:**
    ```bash
    uv tool install openhands --python 3.12
    ```

    **Run OpenHands:**
    ```bash
    openhands
    ```

    **Upgrade OpenHands:**
    ```bash
    uv tool upgrade openhands --python 3.12
    ```
  </Tab>
  <Tab title="Executable Binary">
    Install the OpenHands CLI binary with the install script:

    ```bash
    curl -fsSL https://install.openhands.dev/install.sh | sh
    ```

    Then run:
    ```bash
    openhands
    ```

    <Note>
      Your system may require you to allow permissions to run the executable.

      <Accordion title="MacOS">
        When running the OpenHands CLI on Mac, you may get a warning that says "openhands can't be opened because Apple
        cannot check it for malicious software."

        1. Open `System Settings`.
        2. Go to `Privacy & Security`.
        3. Scroll down to `Security` and click `Allow Anyway`.
        4. Rerun the OpenHands CLI.

        ![mac-security](/openhands/static/img/cli-security-mac.png)

      </Accordion>
    </Note>
  </Tab>
  <Tab title="Using Docker">
    1. Set the following environment variable in your terminal:
       - `SANDBOX_VOLUMES` to specify the directory you want OpenHands to access ([See using SANDBOX_VOLUMES for more info](/openhands/usage/runtimes/docker#using-sandbox_volumes))

    2. Ensure you have configured your settings before starting:
       - Set up `~/.openhands/settings.json` with your LLM configuration

    3. Run the following command:

    ```bash
    docker run -it \
        --pull=always \
        -e SANDBOX_RUNTIME_CONTAINER_IMAGE=docker.openhands.dev/openhands/runtime:1.1-nikolaik \
        -e SANDBOX_USER_ID=$(id -u) \
        -e SANDBOX_VOLUMES=$SANDBOX_VOLUMES \
        -v /var/run/docker.sock:/var/run/docker.sock \
        -v ~/.openhands:/root/.openhands \
        --add-host host.docker.internal:host-gateway \
        --name openhands-cli-$(date +%Y%m%d%H%M%S) \
        python:3.12-slim \
        bash -c "pip install uv && uv tool install openhands --python 3.12 && openhands"
    ```

    The `-e SANDBOX_USER_ID=$(id -u)` is passed to the Docker command to ensure the sandbox user matches the host user’s
    permissions. This prevents the agent from creating root-owned files in the mounted workspace.
  </Tab>
</Tabs>

The first time you run the CLI, it will take you through configuring the required LLM settings. These will be saved
for future sessions.

The conversation history will be saved in `~/.openhands/conversations`.

## Interactive CLI Overview

### What is CLI Mode?

CLI mode enables real-time interaction with OpenHands through the terminal. You can type natural language tasks,
use interactive commands, and receive instant feedback—all inside your terminal.

### CLI Commands and Settings

The OpenHands CLI provides various commands for managing settings, conversations, and configurations. For detailed information about available commands, configuration options, and troubleshooting, see the [CLI Settings](/openhands/usage/run-openhands/cli-settings) page.

Key features include:
- Interactive settings configuration with `/settings`
- MCP server management with `/mcp`
- Conversation management with `/new` and `/help`
- Real-time status monitoring and control

## Command-Line Arguments

### Basic Usage

```bash
openhands [OPTIONS] [COMMAND]
```

### Global Options

| Option | Description |
|--------|-------------|
| `-v, --version` | Show version number and exit |
| `-t, --task TEXT` | Initial task to seed the conversation |
| `-f, --file PATH` | Path to a file whose contents seed the conversation |
| `--resume [ID]` | Resume a conversation. If no ID provided, lists recent conversations |
| `--last` | Resume the most recent conversation (use with `--resume`) |
| `--headless` | Run in headless mode (no UI, requires `--task` or `--file`) |
| `--json` | Enable JSONL output (requires `--headless`) |
| `--always-approve` | Auto-approve all actions without confirmation |
| `--llm-approve` | Use LLM-based security analyzer for action approval |
| `--exit-without-confirmation` | Exit without showing confirmation dialog |

### Examples

```bash
# Start interactive CLI
openhands

# Start with an initial task
openhands -t "Fix the bug in auth.py"

# Resume a specific conversation
openhands --resume abc123def456

# Resume the most recent conversation
openhands --resume --last

# Run in LLM-approve mode
openhands --llm-approve -t "Refactor the database module"
```

## Headless Mode

Headless mode runs OpenHands without the interactive UI, making it ideal for scripting and automation.

### Requirements

- Must specify a task with `--task` or `--file`
- Actions are auto-approved by default

### Basic Headless Usage

```bash
# Run a task in headless mode
openhands --headless -t "Write a Python script that prints hello world"

# Load task from a file
openhands --headless -f task.txt
```

### JSON Output Mode

The `--json` flag enables structured JSONL (JSON Lines) output, streaming events as they occur:

```bash
openhands --headless --json -t "Create a simple Flask app"
```

Each line is a JSON object representing an agent event, useful for:
- CI/CD pipelines
- Automated processing
- Integration with other tools

### Combining Options

```bash
# Headless with JSON output
openhands --headless --json -t "Add unit tests" > output.jsonl

# Headless with LLM-based approval
openhands --headless --llm-approve -f requirements.txt
```

<Note>
For advanced headless usage with Python or Docker directly (without the CLI), see the [Headless Mode](/openhands/usage/run-openhands/headless-mode) documentation.
</Note>

## Subcommands

The CLI provides several subcommands for different functionalities:

| Command | Description |
|---------|-------------|
| `serve` | Launch the OpenHands GUI server using Docker |
| `web` | Launch the CLI as a browser-based web application |
| `cloud` | Create conversations in OpenHands Cloud |
| `acp` | Start Agent Client Protocol server for IDE integrations |
| `mcp` | Manage MCP server configurations |
| `login` | Authenticate with OpenHands Cloud |
| `logout` | Log out from OpenHands Cloud |

### Serve Command

Launch the GUI server locally using Docker:

```bash
# Basic GUI server
openhands serve

# Mount current directory into the container
openhands serve --mount-cwd

# Enable GPU support
openhands serve --gpu
```

### Web Command

Launch the CLI as a web application accessible via browser:

```bash
# Start on default port (12000)
openhands web

# Custom host and port
openhands web --host 0.0.0.0 --port 8080

# Enable debug mode
openhands web --debug
```

### Cloud Commands

Interact with OpenHands Cloud from the CLI:

```bash
# Authenticate with OpenHands Cloud
openhands login

# Create a cloud conversation with a task
openhands cloud -t "Review the codebase"

# Create from a task file
openhands cloud -f task.txt

# Use a custom server URL
openhands cloud --server-url https://custom.server.com -t "Fix bug"

# Log out
openhands logout
```

For more information about cloud features, see [OpenHands Cloud](/openhands/usage/cloud/openhands-cloud).

### MCP Commands

Manage Model Context Protocol servers from the command line:

```bash
# List configured servers
openhands mcp list

# Add an HTTP server with authentication
openhands mcp add my-api --transport http \
  --header "Authorization: Bearer token" \
  https://api.example.com/mcp

# Add a local stdio server
openhands mcp add local-server --transport stdio \
  --env "API_KEY=secret" \
  python -- -m my_mcp_server

# Get server details
openhands mcp get my-api

# Enable/disable servers
openhands mcp enable my-api
openhands mcp disable my-api

# Remove a server
openhands mcp remove my-api
```

For more details, see [CLI Settings - MCP Server Management](/openhands/usage/run-openhands/cli-settings#mcp-server-management).

### ACP Command

Start the Agent Client Protocol server for IDE integrations:

```bash
# Start ACP server
openhands acp

# Resume a conversation in ACP mode
openhands acp --resume abc123def456

# With LLM-based approval
openhands acp --llm-approve
```

For detailed ACP setup with various IDEs, see [Agent Client Protocol](/openhands/usage/run-openhands/acp).
