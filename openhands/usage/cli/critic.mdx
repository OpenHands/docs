## Overview

If you're using the [OpenHands LLM Provider](/openhands/usage/llms/openhands-llms), an experimental **critic feature** is automatically enabled to predict task success in real-time.

For detailed information about the critic feature, including programmatic access and advanced usage, see the [SDK Critic Guide](/sdk/guides/critic).


## What is the Critic?

The critic is an LLM-based evaluator that analyzes agent actions and conversation history to predict the quality or success probability of agent decisions. It provides:

- **Quality scores**: Probability scores between 0.0 and 1.0 indicating predicted success
- **Real-time feedback**: Scores computed during agent execution, not just at completion
- **Iterative refinement**: Automatic follow-up prompts when the critic predicts incomplete work

## Iterative Refinement

When **Iterative Refinement** mode is enabled, the CLI automatically prompts the agent to review and improve its work if the critic predicts a low probability of task success. This creates a self-correcting loop where the agent can catch and fix its own mistakes.

### How It Works

1. The agent completes a task (or calls `FinishAction`)
2. The critic evaluates the result and produces a success probability score (0–100%)
3. If the score falls below the configured **refinement threshold** (default: 60%), a follow-up prompt is automatically sent to the agent
4. The agent reviews its work, identifies remaining issues, and attempts to fix them
5. This process repeats until the score exceeds the threshold or the **max iterations** limit is reached (default: 3)

Refinement can also be triggered when the critic detects **specific issues** (e.g., insufficient testing, loop behavior) above the issue threshold (default: 75%), even if the overall score is acceptable.

### Enabling via Settings UI

1. Open the command palette with `Ctrl+P`
2. Select **Settings**
3. Navigate to the **Critic Settings** tab
4. Toggle on **Iterative Refinement**
5. Optionally adjust the **Refinement Threshold** (1–100%)

### Enabling via CLI

```bash
# Enable with default threshold (60%)
openhands --critic-threshold 0.6 -t "Fix the bug"

# Use a higher threshold (70%) for stricter refinement
openhands --critic-threshold 0.7 -t "Build the feature"
```

Passing `--critic-threshold` automatically enables iterative refinement.

### Configuration Options

| Option | Default | Description |
|--------|---------|-------------|
| **Refinement Threshold** | 60% (`0.6`) | Overall success score below which refinement is triggered |
| **Issue Threshold** | 75% (`0.75`) | Per-issue probability above which refinement is triggered, even if the overall score is acceptable |
| **Max Iterations** | 3 | Maximum number of refinement attempts per user turn (1–10) |

### Example Refinement Prompt

When refinement is triggered, the agent receives a message like:

```
The task appears incomplete (iteration 1/3, predicted success likelihood: 45.0%).

Please review what you've done and verify each requirement is met.
List what's working and what needs fixing, then complete the task.
```

If specific issues are detected, they are included in the prompt:

```
The task appears incomplete (iteration 1/3, predicted success likelihood: 52.0%).

**Detected issues requiring attention:**
- Insufficient Testing (82%)
- Missing Error Handling (76%)

Please review what you've done and verify each requirement is met.
List what's working and what needs fixing, then complete the task.
```

### Status Indicator

When iterative refinement is active, a visual indicator appears in the status bar so you can see that the agent is in a refinement cycle.

## Disabling the Critic

If you prefer not to use the critic feature, you can disable it in your settings:

1. Open the command palette with `Ctrl+P`
2. Select **Settings**
3. Navigate to the **Critic Settings** tab
4. Toggle off **Enable Critic (Experimental)**
