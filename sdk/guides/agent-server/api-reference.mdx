The OpenHands Agent Server exposes a REST and WebSocket API for managing conversations, events, files, and more. Use this API to integrate the agent server into your applications and workflows.

## API Categories

<Columns cols={2}>
  <Card title="Server Details" icon="server">
    Health checks, readiness probes, and server info endpoints for monitoring and orchestration.
  </Card>
  <Card title="Conversations" icon="comments">
    Create, search, pause, resume, and manage agent conversations.
  </Card>
  <Card title="Events" icon="bolt">
    Send messages, search events, and handle confirmation flows within conversations.
  </Card>
  <Card title="Files & Tools" icon="folder-open">
    Upload/download files, manage bash sessions, view git changes, and list available tools.
  </Card>
</Columns>

## Getting Started

To use the REST API, you need a running agent server. See the deployment guides:

- **[Local Agent Server](/sdk/guides/agent-server/local-server)** — Run in the same process for development
- **[Docker Sandboxed Server](/sdk/guides/agent-server/docker-sandbox)** — Isolated Docker containers for production
- **[API Sandboxed Server](/sdk/guides/agent-server/api-sandbox)** — Connect to a hosted agent server via API

Browse the endpoint reference in the sidebar to see full request/response details for each endpoint.
