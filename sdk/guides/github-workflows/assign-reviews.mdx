Automatically assign relevant reviewers to pull requests based on code changes, file ownership, and contributor expertise using intelligent analysis from OpenHands Agent.

## Quick Start

```bash
# 1. Copy workflow to your repository
cp examples/03_github_workflows/04_assign_reviews/workflow.yml .github/workflows/assign-reviews.yml

# 2. Configure secrets in GitHub Settings → Secrets
# Add: LLM_API_KEY

# 3. Configure GitHub Actions permissions
# Settings → Actions → General → Workflow permissions
# Enable: "Read and write permissions"
```

## Features

- **Smart Assignment** - Analyzes code changes and assigns most relevant reviewers
- **File-based Routing** - Considers file ownership and expertise areas
- **Team Integration** - Works with GitHub teams and CODEOWNERS files
- **Automatic Trigger** - Runs when pull requests are opened or updated

## Configuration

The workflow can be customized through environment variables:

```yaml
env:
  MAX_REVIEWERS: 3
  EXCLUDE_AUTHOR: true
  TEAM_ASSIGNMENT: true
```
